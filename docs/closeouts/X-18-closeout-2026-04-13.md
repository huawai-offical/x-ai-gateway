# X-18 关单收口（2026-04-13）

## 原始目标
- 定义协议 DTO 到统一 IR 的映射与校验规范。

## 实现结果
- 提取 4 个聊天协议 mapper：
  - `OpenAiChatCompletionRequestMapper`
  - `OpenAiResponsesRequestMapper`
  - `AnthropicMessagesRequestMapper`
  - `GeminiGenerateContentRequestMapper`
- 聊天 controller 不再内联解析/校验主逻辑，统一收敛到 mapper。
- 新增仓库事实来源文档：
  - `docs/specs/dto-ir-mapping-and-validation-2026-04-13.md`
- 覆盖当前已实现 ingress controller 的 IR 家族分层：`chat execution`、`resource execution`、`direct file`、`direct query / async`

## 验证结果
- 新增 invalid-case 测试：
  - OpenAI Chat 缺少 user message
  - OpenAI Responses 缺少 `function_call_output.call_id`
  - Anthropic 缺少 user payload
  - Gemini 缺少 user payload
- `./gradlew test`：通过

## 遗留边界
- 资源型入口仍以“分类校验 + 资源执行上下文”表达，没有强行引入第二套聊天式 DTO。
